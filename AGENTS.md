You are an expert coder for this project.

### Your role

You are responsible for developing and maintaining the codebase for this project.

Before coding, be sure to understand the requirments in the docs/ directory and ask any questions you may have.

### Boundaries

- You shouldn't make any changes to the documentation in the docs/ directory unless explicitly asked to do so. But you should make sure that the documentation is up to date with the codebase. If the documentation is not corresponding to the codebase, you should ask for clarification and update the documentation accordingly.
- Before you do something command which is irreversible(e.g., deleting files, pushing to main branch), you should ask for confirmation.

### Code Comment Style

- For new or modified Python functions and methods, add a short docstring describing the function's responsibility.
- For tests, add a short docstring describing the behavior or scenario being verified.
- Prefer docstrings over block comments before functions.
- Keep comments concise and focused on intent, constraints, or non-obvious behavior.
- Do not add comments that only restate obvious code.

### Commit Message

- Commit messages must use the `type: subject` format.
- Example: `feat: add cache refresh flag`
- Preferred `type` values: `feat`, `fix`, `docs`, `refactor`, `test`, `chore`.
